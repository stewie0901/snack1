Snake Game Technical Specification (Snake-114 Final)1. 核心機制 (Core Game Mechanics)本專案採用 Grid-based (網格驅動) 邏輯實作。空間定義：地圖尺寸為 $30 \times 30$，透過 DOM 的 data-x 與 data-y 屬性進行精確定位。空間折疊 (Edge Wrapping)：當蛇頭超越邊界時，會從對側出現。計算邏輯：$x_{new} = (x_{current} + \text{GRID\_SIZE}) \pmod{\text{GRID\_SIZE}}$。動態難度 (Dynamic Difficulty)：每當攝取食物時，循環間隔（Interval）減少 2ms，最低限速（Speed Cap）為 10ms。轉向緩衝 (Input Buffering)：引入 nextDirection 變數作為緩衝區，防止在同一幀（Tick）內因快速輸入導致的 180 度「自殺式轉向」。2. 資料結構與變數定義 (Data Structures)變數名稱資料型別功能描述snakeArray<{x, y}>儲存蛇身座標的陣列，snake[0] 為蛇頭（Head）。directionObject {x, y}當前物理移動方向（每一幀執行時的基準）。nextDirectionObject {x, y}使用者輸入後預計改變的方向（緩衝區）。foodObject {x, y}當前食物所在的座標點。speedNumber遊戲循環間隔時間（單位：ms），值越小速度越快。3. 進階邏輯分析 (Advanced Logic)3.1 碰撞偵測 (Self-Collision Detection)利用 JavaScript 的 Array.prototype.some() 方法，檢查新計算的蛇頭座標是否已存在於蛇身陣列中：JavaScriptfunction isCollision(newHead) {
  // 檢查新頭部座標是否與現有蛇身任一節點重疊
  return snake.some(segment => segment.x === newHead.x && segment.y === newHead.y);
}
3.2 渲染流程 (Rendering Pipeline)遊戲採用 「資料驅動渲染」 模式。draw() 函式不負責計算邏輯，僅負責反應數據狀態：Clear: 遍歷所有單元格，移除 .snake 與 .food 類別。Paint Food: 根據 food 座標為對應單元格加入 .food。Paint Snake: 遍歷 snake 陣列，為座標對應的單元格加入 .snake。4. 實作進度總結 (Implementation Progress)[x] 架構規劃 (Planning)：完成開發路徑規劃與模組化設計。[x] 資料管理 (Data Management)：成功使用陣列與座標物件維護遊戲狀態。[x] 地圖繪製 (Rendering)：結合 CSS Grid 與 Data Attributes 實現高精度渲染。[x] 移動控制 (Movement)：實作具備安全保護機制（防反向）的方向控制。[x] 食物系統 (Food System)：實作隨機生成與吃到食物後自動加速機制。[x] 死亡判定 (Game Over)：精確偵測蛇身自撞。[x] UI 綁定 (UI Binding)：完成 Start 與 Pause 的狀態機切換邏輯。5. 逆向工程心得 (Learning Insights)DOM 與數據解耦合 (Decoupling)：學會將 snake 陣列視為「單一事實來源 (Single Source of Truth)」，DOM 僅是資料的視覺化呈現，這讓邏輯除錯變得簡單。計時器優化 (Loop Optimization)：學會透過動態清除並重新設定 setInterval，在不重新整理頁面的情況下平滑地改變遊戲節奏。防禦性程式碼 (Defensive Programming)：透過 nextDirection 的設計，體會到如何從程式碼層面預防使用者非預期操作（如快速連按）導致的 Bug。
